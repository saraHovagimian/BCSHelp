Preproceso

Archivo

Es posible indicar un archivo para preproceso del archivo a importar, con extensión awk, que se aplicará automáticamente sobre el archivo a importar antes de iniciar el proceso de importación. Se dispone de un botón para buscar dicho archivo. Para que este proceso funcione es preciso que el archivo awk.exe se ubique en directorio Programs de la instalación de BASCS. Vale aclarar que AWK es un lenguaje de programación diseñado para procesar datos basados en texto. Esta funcionalidad puede utilizarse en los casos en que el archivo a importar tiene un formato muy especial y debe adecuarse antes de la importación.

Es posible no eliminar, al finalizar el proceso de importación, los archivos intermedios generados al utilizar un archivo de preproceso, o en la eliminación de caracteres extraños del archivo original.

Para ello podrá definirse una variable ambiental de usuario con nombre NOBORRARARCIMP, que en caso de crearse indicando que tiene como valor 1, provoca que el sistema no elimine los archivos mencionados al finalizar el proceso de importación.

Los archivos se podrán encontrar en el directorio de instalación de BASCS o en la ubicación del archivo de importación. El correspondiente a la aplicación del archivo de preproceso tiene el prefijo ~PRE y la extensión TXT. El correspondiente a la eliminación de caracteres extraños tiene el mismo nombre que el archivo de importación y la extensión BAS.

En caso de no existir la variable o no tener valor 1, dichos archivos se eliminarán al finalizar el proceso de importación, tal como ocurría hasta ahora.

Eliminar coeficientes anteriores a mesesEliminar coeficientes anteriores a meses

Es obligatorio informar que se eliminen coeficientes anteriores, para evitar resguardar innecesariamente la información de los padrones de Ingresos Brutos que generalmente tienen un tamaño considerable y pueden ocasionar un crecimiento de la base de datos.

El valor default será de 3 meses permitiendo modificarlo. Si se informa resguardar una cantidad mayor de meses se advertirá tal situación, permitiendo continuar igualmente con la importación.

 Procesar por lotes de registros Procesar por lotes de registros

Es posible realizar el proceso de importación dividiendo la información a procesar por lotes, indicando la cantidad de registros a procesar por vez, o en forma conjunta. El proceso no puede interrumpirse hasta que se finalice con el procesamiento de todos los registros. El hacerlo por lotes, en algunos equipos, puede mejorar el uso de los recursos de los mismos. Por default, se ofrece procesar de a 10000 registros

Generación de archivo de logGeneración de archivo de log

Es posible generar un archivo de log que permitirá ver los lugares por donde va el proceso de importación de padrones.

Para poder generar este archivo se debe configurar un parámetro especial oculto, llamado LOGIMPORTACIONPADRONES.

Si se tiene ese parámetro indicado con valor 1, se va a generar durante la importación un archivo de log con nombre LOGIMPPADRONddmmaa, siendo ddmmaa la fecha de la importación.

El archivo se va a generar en la carpeta BASCS dentro de la carpeta de acceso público de todos los usuarios. Si esa carpeta no existe es creada al crear el archivo de log.

Para comenzar a generar ese archivo se debe ejecutar la siguiente instrucción en la base de datos:

insert into parametroscod (codparam , paramint ) values ('LOGIMPORTACIONPADRONES', 1)

Si en algún momento se quiere dejar de generar dicho log, se debe eliminar el registro de parametroscod haciendo:

    DELETE FROM parametroscod WHERE codparam='LOGIMPORTACIONPADRONES'

O bien,  setearle el valor en 0 al parámetro:

   UPDATE parametroscod set paramint=0 WHERE codparam='LOGIMPORTACIONPADRONES'

 

Pasado un tiempo es necesario  eliminar los archivos antiguos generados por este proceso .Por lo cual,s i en el momento de realizar una importación de padrones, se detectan archivos de este tipo con más de 3 meses de antigüedad, éstos se eliminarán..

Para tener en cuentaPara tener en cuenta

Si el proceso detecta que se está reemplazando información correspondiente a una misma fecha emite el Aviso Ya existe información importada para la fecha de publicación dd/mm/aaaa en la base de datos ¿Desea continuar?. De modo que el usuario decida si desea seguir adelante, o no.

Al finalizar el proceso de importación, el programa informa al usuario del resultado y la cantidad de registros importados.

Debe tenerse en cuenta que el padrón es muy grande por lo que el proceso de importación podría llevar mucho tiempo. Durante la importación se muestra una barra color azul en la parte inferior de la pantalla, que permite ver el porcentaje de avance del proceso.
