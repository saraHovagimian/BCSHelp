Tipos de documentos

Acá se pueden definir los diferentes tipos de documentos y de números impositivos con los que se opera.

Los mismos podrán asociarse a clientes , proveedores , agentes , sucursales y bancos que se ingresen, dependiendo de la categoría de cada uno frente a la D.G.I.

Los datos que deben ingresarse son los siguientes:

CódigoCódigo

Código de hasta tres caracteres que identifica al tipo de documento y será utilizado para individualizar al mismo en todo el sistema.

BAS CS se entrega con algunos tipos de documentos ya definidos. Estos tipos son:

00 C.I. POLICIA FEDERAL

80 C.U.I.T.

89 L.E.

90 L.C.

94 PASAPORTE

95 D.N.I.

DescripciónDescripción

Denominación del tipo de documento.

Se dispone de hasta veinticinco caracteres para el ingreso.

Método de validaciónMétodo de validación

Número de un dígito que identifica el método de validación que se utilizará para verificar los números de documento que se ingresen del tipo indicado.

Si se indica 1 en este campo el documento se validará como C.U.I.T.

El tipo de validación que se desea efectuar para cada tipo de documento podrá ser programado por el usuario según las necesidades de cada caso.

Para esto se deberán definir en la base de datos, para cada método de validación, procedimientos denominados Stored Procedures que se deberán llamar SP_VALIDADOCn, donde n es el tipo de validación de los documentos para el que se desea efectuar determinados controles para determinar la validez de dichos documentos.

El procedimiento asociado a un método de validación se ejecutará automáticamente al ingresar en el sistema un nuevo número de documento con esa validación. Si el procedimiento devuelve un 0, significa que el número de documento es válido, en caso contrario el número de documento es inválido.

SP_VALIDADOC1, que permite validar los números de C.U.I.T., ya viene con el sistema.
