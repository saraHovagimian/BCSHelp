Fechas, avance y operaciones de la tarea

En este menú contextual se exhiben diferentes fechas relacionadas al inicio y fin de cada una de las tareas, grado de avance y la opción de generar partes parciales en relación a cada una de ellas.

Fechas

  • Fechas estimadas

Las “fechas y horas estimadas de inicio y finalización” de una tarea corresponden a las que se han indicado al registrar la orden de producción.

  • Fechas previstas

Si la tarea aún no está iniciada o terminada, además de las fechas estimadas (de inicio y fin), posiblemente se muestren también las “fechas previstas de inicio y finalización” de la tarea.

Las fechas previstas se exhiben cuando algunas de las tareas de las cuales ella depende, empezaron o terminaron en fechas diferentes a las estimadas. En ese caso se recalculan las nuevas fechas estimadas (de inicio y fin) y se exhiben con el nombre de fechas previstas.

  • Fechas reales

Las “Fechas reales de inicio y terminación” corresponden a las fechas en las que la tarea efectivamente se ha iniciado y terminado.

Es decir, si la tarea está iniciada se muestra la Fecha y hora real de inicio y, si está terminada, se muestra información sobre la fecha y hora real de finalización.

Las operaciones a realizar sobre las tareas dependen del estado en que éstas se encuentran. Sobre las tareas no iniciadas, o que pueden iniciarse, es posible ejecutar "Iniciar tarea" y "Terminar tarea". Sobre las tareas que están iniciadas se puede ejecutar "Terminar tarea" y realizar partes parciales.

Esta línea muestra el porcentaje de avance de la tarea que se está consultando.

Parte parcial

Es posible generar partes parciales en relación a cada una de las la tareas.

Se debe tener en cuenta las siguientes consideraciones:

  • Sólo se pueden generar partes parciales cuando la tarea está iniciada.

  • se puede hacer un parte parcial de una tarea si aquellas de las que depende no están iniciadas.

  • Si las tareas de las que depende están iniciadas, si se registró algún parte parcial de las mismas y si se superó el porcentaje de avance indicado en la solapa de dependencias para la tarea en cuestión, la misma puede iniciarse. De esta manera, pueden existir varios partes de producción para una misma tarea.

Si se genera un parte parcial de una tarea que involucra varios grupos de trabajo, se pedirá el grupo antes de ingresar al comprobante del Parte. Una vez informado el grupo, al registrar el parte parcial en la “Ficha Productos” se debe informar la cantidad que realiza ese grupo en relación a la tarea de la que se trate.

Se ofrece como producto en proceso del parte parcial, la suma de las cantidades de la tarea anterior menos lo informado en los partes parciales para la tarea que se está ingresando el parte.

Para el caso de productos terminados, el algoritmo que utiliza el sistema para calcular la cantidad de producto terminado que se ofrece en los partes de producción (para una tarea que permite grupos de trabajo) es el siguiente:

Primero se divide la cantidad esperada para la tarea por la cantidad de grupos de trabajo. Por otro lado se le resta a la cantidad esperada para la tarea lo hecho por todos los grupos de trabajo para esa tarea. Luego se selecciona el menor de esos dos valores y se resta lo que ya hizo el grupo para esa tarea. Si el valor resultante es mayor a cero, ese es el valor que se ofrece. Si es cero o negativo, se ofrece la cantidad pendiente de la tarea.

También es posible anular un inicio de tarea y una terminación de tarea, siempre y cuando la tarea en cuestión esté iniciada o terminada, y que sea la última tarea terminada o iniciada en el árbol de dependencias al que pertenece.

Al ejecutar la tarea "Iniciar tarea", se habilita una ventana en la que se muestran los datos de la agenda estimada de la tarea y se permite el ingreso de la fecha y hora real de inicio de la misma y la fecha y hora prevista de terminación.

El sistema controla que la fecha de inicio real que se indica no sea posterior a la fecha actual y que la fecha y hora de inicio real no sean posteriores a la fecha y hora de finalización estimada (o prevista).

Informado el inicio, al oprimir el botón "Grabar", se modifica el gráfico correspondiente a las tareas de la orden de producción, teniendo en cuenta los cambios ocasionados por la tarea recién iniciada.

Al ejecutar la tarea "Terminar tarea", se habilita una ventana donde se muestran los datos de la "agenda prevista" de la tarea y se permite el ingreso de la fecha y hora real de inicio de la misma (ya que si la tarea aun no está iniciada, se permite informar el inicio conjuntamente con su terminación) y la fecha y hora real de terminación.

El sistema controla que la fecha real de finalización que se indica no sea posterior a la fecha actual y que la fecha y hora de inicio real no sean posteriores a la fecha y hora de finalización real.

Al oprimir el botón Grabar (F10), se emite un aviso preguntando al usuario si se quiere generar el parte de producción correspondiente.

Si se opta por generar el parte de producción y el cierre de la tarea en cuestión requiere que deban terminarse otras tareas de las cuáles depende, y que aun no generaron el correspondiente parte de producción, el sistema muestra una tabla con las tareas terminadas para que el usuario seleccione cuáles de ellas desea incluir en el parte de producción parcial a generar.

Este parte de producción parcial incluirá los insumos y recursos pendientes correspondientes a las tareas que se terminaron y se seleccionaron para la generación del mismo. Dichos insumos serán dados de baja del stock.

Cuando al terminar una o más tareas se hace un parte parcial y no se consumen todos los insumos de las tareas que se cierran, el sistema emite un mensaje preguntando si se desean cancelar los insumos pendientes de las correspondientes tareas.

Al finalizar una tarea queda como cantidad a producir para las siguientes tareas la suma de las cantidades informadas en la tarea en cuestión.

También, cuando se está haciendo un Parte parcial en relación a una orden de producción que incluye un solo producto terminado y se modifica la cantidad, se ofrece recalcular los insumos.
