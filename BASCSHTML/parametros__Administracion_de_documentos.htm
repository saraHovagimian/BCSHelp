Administración de Documentos

Permite adjuntar documentos externos, previamente digitalizados, a comprobantes del sistema (ordenes de compra, pedidos, comprobantes de compra y venta, carpetas de importación, etc).

Esta funcionalidad esta disponible a partir de la versión BASCS XE 4.3.1.

Debe ser configurada por un usuario que pertenezca al role de servidor SYSADMIN, un usuario SA u otro usuario con el rol de SA.

La configuración finaliza grabando esta pantalla. BASCS en conjunto con SQL realiza la configuración interna necesaria para que funcione correctamente el adjuntar y guardar documentos.

Una vez configurado, en la parte superior de la pantalla se podrá observar que la funcionalidad esta activada.

Carpeta para Almacenamiento

Debe indicarse la carpeta en la cual se guardarán los documentos adjuntos, ésta estar ubicada en el servidor en el cual se encuentra la base de datos.

Por defecto, BASCS XE ofrece la ruta en la cual se aloja la BD, agregando una barra “/” al final de dicha ruta. Luego de la misma, el usuario administrador debe indicar un nombre para denominar la carpeta en la cual SQL guardará los documentos adjuntos. Esta carpeta no debe existir en dicho ruteo pues será creada por SQL en la configuración que realiza.

Usuario

Como se sabe, hay dos maneras de dar de altar usuarios de BAS CS: con seguridad integrada o como usuarios de SQL.

Si los usuarios de BAS CS ingresan con seguridad integrada puede obviarse indicar el usuario y su password en este apartado. Pero, si hay usuarios que acceden a la base de datos y se han definido como usuarios de SQL, para poder grabar en la carpeta indicada es necesario definir, en el cuadro Usuario, un usuario del dominio (y su clave) para que pueda acceder a la carpeta y grabar. Esto se debe a que el mecanismo implementado por SQL Server requiere trabajar con seguridad integrada; de este modo, si el usuario que quiere grabar un documento es un usuario de SQL, el usuario que efectivamente realizará la grabación es el que aquí se indique.

Tamaño Máx. Archivo

Permite indicar un tamaño máximo permitido del archivo a adjuntar. Si se intenta ingresar en el sistema un documento con un tamaño superior al indicado el sistema no permitirá su ingreso avisando que su tamaño supera el máximo permitido.

Por default esta configurado en 20MB, pudiendo ser modificado. En caso de dejar en blanco la opción, se podrá adjuntar cualquier tamaño de archivo.

Deshabilitar

Para deshabilitar la función de Adjuntar documentos, simplemente se debe apretar el botón Deshabilitar ubicado en la parte inferior izquierda de la pantalla.

Al presionarlo se eliminan de la base de datos todos los seteos realizados para la administración de documentos.

Es sumamente importante conocer que esta deshabilitación, elimina todos los archivos adjuntados siendo imposible volver a recuperarlos y/o asociarlos a los comprobantes originales.

La siguiente información se brinda solamente a título informativo y no es requerida para implementar esta funcionalidad.

Como resultado de la habilitación de la administración de documentos realizada, en el SQL se activan los siguiente parámetros:

En las propiedades del Server sobre el cual se esta trabajando, opción Advanced, en Filestream Access Level se setea la opción Full Access enabled. Independientemente de la opción que haya estado configurado en este punto, BASCS lo modifica y lo configura con acceso total.

Si bien esta opción se habilita al configurar la administración de documentos desde BAS CS, si ésta se deshabilita, el parámetro no se modifica. De este modo se asegura que, si estaba siendo utilizado por otra aplicación, ésta no se vea afectada.

Vale aclarar que este mecanismo se habilita por servidor y no por base de datos. Una vez establecido queda habilitado para todas las bases de datos del servidor.

En las propiedades de la base de datos sobre la cual se está trabajando, Opción Filesgroup, en la parte inferior de la ventana, bajo la denominación Filestream, debe existir un documento llamado BASDOCUMENTOS, que es denominación de la carpeta virtual en al cual se guardaran los documentos adjuntados.

En las propiedades de la base de datos sobre la cual se está trabajando, Opción Files, debe existir un tercer file denominado BASDOCUMENTOS. Este tercer file se une a los ya existentes, log y data.

La existencia de una tabla en la base de datos que se llama BASDOCUMENTOS. La misma es creada en la instalación, por lo que su existencia denota que el proceso corrió correctamente. En caso de deshabilitar el funcionamiento, esta tabla desaparece.
