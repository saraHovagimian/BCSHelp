Formatos de Exportación

Desde este punto se puede acceder a los formatos estándar (están predefinidos)que se utilizan para exportar archivos a los utilitarios impositivos o crear nuevos formatos de importación. Se debe tener en cuenta que los formatos estándar se pueden modificar pero no eliminar.

Todos los utilitarios impositivos generan archivos de texto cuyos formatos son diseñables por el usuario. BAS CS dispone de formatos estándar y en los campos Formato de cada utilitario se ofrecen los códigos de los mismos. No obstante, estos pueden modificarse según se requiera.

Tipo de formatoTipo de formato

Se debe indicar el tipo de formato. Los tipos de formatos viene predefinidos en el sistema y pueden consultarse presionando la trecla F3. El usuario no puede agregar tipos.

FormatoFormato

Código identificatorio del formato que se define.

Es un campo alfanumérico de hasta cinco dígitos y permitirá identificar al formato en todo el sistema.

Descripción Descripción

Denominación del formato. Se dispone de hasta 80 caracteres.

ParametrizaciónParametrización

Formato

Los formatos pueden ser de dos tipos de acuerdo a la forma en que se detallan los campos en los registros:

C.S.V. (valores separados por comas)

En este caso los valores se separan por comas por lo que se debe indicar el orden (posición) en el registro de los campos que se detallen.

Posición fija

Si se indica esta opción, los valores se ubican a partir de la columna que se indique (inicio) ocupando tantos caracteres / dígitos como se indique (largo). Los mismos se completarán con ceros o espacios a derecha o izquierda de acuerdo a lo que se establezca en el aplicativo para cada valor.

Longitud registro

En el caso de que el formato sea de Posición fija es conveniente que se complete el dato Longitud de registro. En caso que no se informe, la longitud del registro se determina analizando los datos inicio y largo de cada uno de los campos para determinar la columna mayor (es la mayor que resulta de adicionar al dato Inicio de cada campo la longitud que le corresponde). Si este valor resulta inferior al indicado por el aplicativo en cuestión, se considera como longitud del registro el del aplicativo.

Campos

En esta grilla se define la estructura de los registros del archivo a exportar.

Si se activa el campo Posición fija se deben completar las siguientes columnas.

Campo

En esta columna se debe seleccionar un campo de los disponibles para el tipo de formato. Cada tipo de formato tiene campos propios y varían de un tipo a otro. Estos campos ya vienen predefinidos y el usuario no puede agregar o eliminar campos.

Para cada tipo de formato hay campos obligatorios, por lo cual si se intentase definir un formato sin incluir alguno de dichos campos, un mensaje informará tal situación.

En cada formato se podrán repetir todos los campos que se requieran. Cuando se seleccionan se permite elegir varios en forma simultanea.

Inicio

Columna en la que inicia el campo.

Largo

Longitud (en columnas) del campo.

Formato fecha

Cuando se selecciona un campo Fecha se debe indicar el formato de la fecha deseado.

Texto

Entre los campos que pueden incluirse en el formato se prevé la posibilidad de informar textos libres ingresados por el usuario. Luego de indicar un campo Texto es posible indicar el texto deseado en esta columna.

Atributo C.

Código de un atributo de cliente.

Permite incluir un dato en el archivo que no ha sido previsto en BAS CS.

Descripción.

Descripción del atributo anterior.

Atributo P.

Código de un atributo de proveedor.

Permite incluir un dato en el archivo que no ha sido previsto en BAS CS.
