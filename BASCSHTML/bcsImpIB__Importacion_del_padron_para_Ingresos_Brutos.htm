Importación del padrón para Ingresos Brutos

Este utilitario permite incorporar al sistema un padrón destinado al cálculo de percepciones y retenciones de ingresos brutos para provincias que tienen cálculo directo a partir del archivo correspondiente.

Formato

Este campo ofrece el código del formato estándar previsto por BAS CS para generar el archivo. Pueden consultarse todos los formatos disponibles pulsando F3.

Si se desea acceder a las características y al detalle de un formato se debe presionar la tecla F4 estando ubicados en el campo formato que se quiere consultar o definir. Debe tenerse en cuenta que los formatos estándar se pueden modificar pero no eliminar.

Para tener en cuanta

Los formatos estándar que ofrece BAS CS pueden modificarse. Para hacerlo se puede consultar en formatos de exportación.

Nombre del archivo Examinar

Se debe indicar el nombre del archivo a importar, puede buscarse mediante el botón “Examinar”.

Padrón

Código del padrón a importar.

El padrón que se indique debe haberse dado de alta con anterioridad al proceso de importación.

Reemplazar datos existentes

Tildar para indicar que se desea reemplazar los datos existentes.

Solo reemplazar los registros que hayan cambiado

Tildar si sólo se desea reemplazar los registros que hayan cambiado.

Eliminar coeficientes anteriores a meses

Tildar si se desea eliminar coeficientes anteriores e indicar la cantidad de meses a eliminar.

Procesar por lotes de registros

Es posible realizar el proceso de importación dividiendo la información a procesar por lotes, indicando la cantidad de registros a procesar por vez, o en forma conjunta. Ya sea que se trabaje por lotes, o no, el proceso no puede interrumpirse hasta que se finalice con el procesamiento de todos los registros. El hacerlo por lotes, en algunos equipos, puede mejorar el uso de los recursos de los mismos.

Para tener en cuenta

  • Si el proceso detecta que se está reemplazando información correspondiente a una misma fecha emite el Aviso “Ya existe información importada para la fecha de publicación dd/mm/aaaa en la base de datos ¿Desea continuar?”. De modo que el usuario decida si desea seguir adelante, o no.

  • Al finalizar el proceso de importación, el programa informa al usuario del resultado y la cantidad de registros importados.

  • En caso que se encuentren datos que no se hayan podido importar por presentar alguna anomalía, ya sea por porcentajes diferentes para el mismo CUIT u otros problemas, el detalle quedará registrado en un archivo de nombre “BCSImpIB.log”.

  • Debe tenerse en cuenta que el padrón es muy grande por lo que el proceso de importación podría llevar mucho tiempo.
