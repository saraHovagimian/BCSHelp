Ingreso de Ítems a la Modificación de inventario

Para ingresar los datos en este comprobante, se accede a una pantalla intermedia denominada “Ingreso” al hacer click sobre esta tabla, estando la misma vacía, u oprimiendo el botón derecho del mouse sobre dicha tabla, que hablita un menú contextual con las opciones “Nueva” o “Insertar”.

Mediante esta tabla se informan los bienes a incluir en el comprobante de modificación del inventario que se está confeccionando.

Sólo será necesario ingresar ítems en esta tabla si se afectan nuevos bienes al inventario indicado.

Para cada ítem se debe informar la cantidad que se afecta y, en función al tipo de ítem del que se trate, se requerirá información adicional.

Al respecto, y a título de ejemplo, podemos mencionar que para algunos bienes, si la mercadería que egresa ingresó avalada por un despacho de importación, éste deberá indicarse en el comprobante de afectación. En ese caso, además, el sistema lleva un control estricto del stock disponible por despacho. En caso de bienes tales que cada unidad disponible en stock tenga asociado un número de serie en particular, cada unidad que egresa deberá identificarse, también, por su respectivo número de serie.

Es importante tener en cuenta que la forma en que el sistema solicita los datos a incluir en el comprobante varía según cual sea el tipo de bienes de los que se trate. En algunos casos, como por ejemplo cuando se trabaja con doble unidad de medida, se abre una ventana en el campo Cantidad para requerir los datos necesarios.

En otros casos, como por ejemplo, cuando deben ingresarse códigos de despachos o partidas, se incorporan columnas adicionales para permitir incluir dicha información.

Los datos a incluir en la pantalla intermedia “Ingreso” son:

Ficha Ítem

Ítem

Código del ítem a incluir en el comprobante.

Debe corresponder a un bien o servicio ingresado en el sistema.

Puede ingresarse el código del ítem o servicio o su código alternativo.

Descripción

Descripción del ítem indicado en el dato anterior.

Si el ítem tiene asociada una descripción larga, se exhibirá esta. Si no, se exhibirá la otra descripción. Si el ítem no tiene descripción asociada, se solicitará el ingreso de un detalle del mismo en este momento.

En el caso que se trate de bienes con colores y medidas es necesario ingresar el código del color y medida junto con el código principal del bien. Si hablamos de bienes con contenedores, no será necesario indicar el contenedor.

Los bienes con dimensiones se ingresan al sistema con dimensiones 1x1x1.

Cantidad

Cantidad de unidades que egresan del stock y que se afectan del ítem indicado.

Aquí se presentan varias posibilidades.

Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las compras se registran en la primera unidad. Si las compras se registran en segunda unidad, al ingresar la cantidad en la segunda unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable.

Si el ítem trabaja con dimensiones, se abrirá una ventana que solicitará el ingreso de cantidades para cada dimensión posible definida para el ítem, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por todas las dimensiones del ítem.

Cantidad en 2da unidad

Cantidad de unidades que se afectan del ítem indicado, en la segunda unidad de medida.

Si el ítem no trabaja con doble unidad de medida, no se solicitará información en este campo.

Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las compras se registran en la segunda unidad. Si las compras se registran en primera unidad, al ingresar la cantidad en la primera unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable.

Importe

El importe del bien o servicio si es de tipo importe. Cabe aclarar que es posible ingresar un valor con signo negativo. Esto se utiliza para reflejar una depreciación en el bien.

Negocio

Si se ingresan centros de apropiación y alguno de los centros indicados administra negocios se debe ingresar el código del negocio asociado a la línea del comprobante.

Debe indicarse el código de un negocio ya definido en el sistema y correspondiente a la empresa en la cual se ingresa el comprobante.

Despacho

Código de hasta quince caracteres que indica el número de despacho del cual egresa el bien indicado.

Este dato podrá ingresarse para los ítems que trabajan con despachos. Debe corresponder a un despacho definido en el sistema y debe tener disponible la cantidad de unidades del ítem que se está entregando.

El sistema muestra una lista ordenada por fecha con todos los despachos en los que ingresó el bien y podrá seleccionarse el despacho que corresponda.

Partida

Código de hasta quince caracteres que indica el número de partida de la cual egresa el bien indicado.

Este dato podrá ingresarse para los ítems que trabajan con partidas. Debe corresponder a una partida definida en el sistema para el ítem indicado.

El sistema mostrará una lista con todas las partidas en las que ingresó el bien y podrá seleccionarse la partida que corresponda.

Números de serie

Si el bien indicado trabaja con números de serie, se abrirá una ventana que permitirá el ingreso de todos los números de serie que egresan con el comprobante. Sólo se podrán ingresar bienes que administre números de serie si manejan cantidades unitarias.

Se solicitará el ingreso de tantos números como cantidad de unidades egresen del ítem.

Se brinda la posibilidad de ingresar estos números en forma automática definiendo un prefijo común y un rango de números para los mismos. Esto es muy útil en el caso de egresos de grandes cantidades de mercadería seriada. Además, se exhibirá una lista con todos los números de serie registrados en los ingresos de mercadería del ítem, permitiendo también seleccionar de estos números cuáles son los que egresan.

Botones adicionales para navegación en la pantalla de ingreso

Los botones “Anterior” (AvPág) y “Siguiente” (RePág) permiten agregar filas a la pantalla de inventarios (al final de esta grilla) o recorrer las ya existentes hacia delante y hacia atrás. También se dispone de botones para ir directamente a la fila inicial (Primero) y a la final (Ultimo).

Ficha Observaciones

Observaciones del bien o servicio , en el caso que correspondiere.
