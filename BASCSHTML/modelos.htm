Modelos

En todas las ventanas de trabajo del sistema es posible definir "Modelos".

Suele ocurrir que, ya sea para registrar comprobantes, emitir informes o realizar procesos, las pantallas se deben completar de la misma manera una y otra vez.

Los modelos permiten dejar precargados una serie de campos de una pantalla con determinados datos de modo que no tener que hacerlo cada vez que se accede a la misma. Es posible definir varios modelos para una misma ventana de trabajo.

Al definir un modelo, cualquiera sea la ventana del sistema, se debe indicar qué usuarios y/o qué grupo de usuarios podrán utilizarlo, modificarlo y/o eliminarlo.

En base a esta información, cuando un usuario ingrese a la ventana correspondiente a un programa, verá solamente los modelos que se hayan habilitado para él, ya sea que se hayan habilitado específicamente para él, para el grupo o grupos al cual pertenece o para todos los usuarios.

Además, se podrá indicar un modelo por defecto para cada usuario o grupo de usuarios, en relación a cada ventana, de modo que éste se precargue automáticamente cuando se ingrese a la correspondiente pantalla.

Un ejemplo de uso de los modelos se puede encontrar en la definición de clientes. Al darlos de alta, hay una serie de datos que se repiten, el campo categoría de IVA se debe completar como "Inscripto", el campo destinado al país será, en general, Argentina y así, se encontrarán una serie de datos comunes a asignar a conjuntos de clientes. En ese caso se podrá definir un modelo que tenga esos campos precargados, asignándole un nombre, por ejemplo, Clientes inscriptos Argentina. Así, podrán definirse todos los modelos de carga de clientes que se requieran.

Para definir un modelo se debe completar la pantalla con los datos que se requieren en cada uno de los campos y luego presionar el botón (Figura 1-Botón Modelos).

Figura 1-Botón Modelos

Al presionar el botón de Modelos se abre la ventana destinada a definir, modificar y borrar los modelos correspondientes a esa ventana de trabajo (Figura 2-Definición de Modelos).

Figura 2-Definición de Modelos

Todos los modelos que habilitados para el usuario que está utilizando el sistema, en relación a una ventana, se exhiben en el encabezado de la misma (Figura 3-Modelos definidos).

Figura 3-Modelos definidos

Para que un usuario pueda dar de alta modelos en una ventana del sistema debe tener el correspondiente permiso. La parametrización se realiza en la ficha Modelos de la aplicación Permisos por programa (Figura 4-Permisos por programa).

Figura 4-Permisos por programa

El campo No permite dar de alta modelos permite indicar para el usuario, o para el grupo de usuarios del cual se trate, si tiene permiso o no para dar de alta modelos en el programa que se indica en el campo Programa (Figura 4-Permisos por programa). Por defecto, el campo se presenta deshabilitado. Es decir que, si no se indica lo contrario, todos los usuarios y grupos que se definan están habilitados para definir modelos. Para que no puedan hacerlo se deberá indicar explícitamente lo contrario tildando dicho campo.

Si el usuario no tiene permiso para dar de alta modelos, al pretender grabar un modelo, el botón Grabar de la Figura 2-Definición de Modelos se presenta deshabilitado.

Luego volveremos sobre los permisos por programa para analizar en detalle las Parametrizaciones previstas en la Figura 4-Permisos por programa. Ahora nos detendremos en los detalles relativos al alta y modificación de modelos (Figura 2-Definición de Modelos).

Ya sea que se esté dando el alta de un nuevo modelo o se esté modificando uno existente, al presionar el botón de Modelos se accede a la ventana de la Figura 2-Definición de Modelos.

El campo Descripción permite indicar el nombre del modelo y el campo Comentario ampliar, brindando más detalles, la información del modelo que se define.

La Descripción que se informe se exhibe en la Figura 3-Modelos definidos y el texto del Comentario aparece como un tool-tip cuando se acerca el cursor al nombre del modelo (Figura 5-Comentario del modelo).

Además, en el campo Descripción, pueden consultarse todos los modelos definidos para la ventana, de modo de seleccionar el que se requiera.

Figura 5-Comentario del modelo

El campo Modelo predeterminado (Figura 2-Definición de Modelos) permite indicar si el modelo que se define corresponde al modelo por defecto.

Esto se debe a que, obligatoriamente, en caso de definir modelos debe indicarse a uno de ellos como modelo Por defecto". De este modo, cada vez que se ingrese a la tarea, se precargarán los campos de la pantalla con la información que figure en el modelo que se ha indicado como "Modelo predeterminado".

El campo Modifica los valores predeterminados guardados en el modelo (Figura 2-Definición de Modelos) está destinado a los casos de modificación de modelos. Si el campo se tilda se modifican los valores guardados en el modelo con la información que se encuentra registrada en la ventana correspondiente al programa.

Por lo cual si se tiene registrada información en la ventana correspondiente a un programa y se accede al mantenimiento de modelos, al presionar Grabar, la información del modelo se modificará (o no), en base a lo que se indique en el campo Modifica los valores predeterminados guardados en el modelo.

En caso de dar de alta un nuevo modelo este campo aparece deshabilitado.

En caso de acceder a un modelo existente, por defecto, el campo se presenta habilitado y desactivado (sin tildar) como se muestra en la Figura 2-Definición de Modelos.

El botón Seguridad (Figura 2-Definición de Modelos) está destinado a que el usuario defina con quienes va a compartir el modelo y que nivel de permisos les otorgará. Al presionarlo, se accede a la ventana de la Figura 6-Seguridad.

Al dar de alta un modelo, en dicha ventana se precarga una línea indicando el usuario que creó el modelo y asignándole permisos para uso, modificación y baja. Esta línea no se puede borrar.

El usuario puede agregar otros usuarios y/o grupos e indicar, para cada uno, que permisos le otorgará. También podrá eliminarlos cuando lo requiera; para ello bastará eliminar las filas correspondientes a los usuarios de los cuales se trate.

Es posible otorgar a los usuarios sólo permiso para utilizar los modelos, o para modificarlos e, incluso, eliminarlos.

La ventana exhibe el nombre del usuario que creó el modelo y la fecha de creación del mismo. También se exhibe el nombre del usuario que realizó la última modificación y la fecha correspondiente. Cuando se crea un modelo nuevo, estos datos se presentan en blanco, su información se completa al presionar el botón Grabar (Figura 6-Seguridad).

Si un usuario da de alta un modelo sin ir a la ventana Seguridad, el sistema le asigna permiso de uso, modificación y baja al usuario que lo crea y al grupo TODOS (sin asumir que es el modelo por Defecto) (Figura 6-Seguridad). Al usuario que crea el modelo se le asigna como modelo por defecto el que figure como Modelo predeterminado.

Por lo cual, al definir un modelo se asigna permiso de uso a todos los usuarios pero, bastará ingresar a la ventana de Seguridad para que sólo pueda utilizarlo el usuario que lo crea y los que él designe.

Figura 6-Seguridad

Respecto a la administración de modelos, es importante tener en cuenta que:

 Los permisos que tiene un usuario sobre un modelo resultan de la suma de los permisos que tiene dicho usuario para el modelo, más los permisos que tienen los grupos a los cuáles pertenece para el modelo en cuestión. Por ejemplo si un usuario no tiene permiso para grabar un modelo, pero el grupo sí lo tiene, el usuario tendrá permiso para grabar el modelo.

 Si el usuario no tiene permiso para dar de baja el modelo, se deshabilita el botón Borrar

 Si el usuario no tiene permiso para dar de alta modelos, se deshabilita el botón Grabar para nuevos modelos

 El check Modelo predeterminado estará habitado sólo cuando se da de alta un nuevo modelo

 Si el usuario no tiene permiso de modificación, se deshabilitan los campos Comentario, Modelo predeterminado, el botón Grabar y el botón Seguridad.

Por último, vale aclarar que los modelos que se muestran al usuario, en relación a un programa, serán todos los que se hayan habilitado para el usuario, para el grupo o grupos a los cuales pertenece dicho usuario y los que se hayan habilitado para todos los usuarios. El modelo que se mostrará como predeterminado (por defecto) será el que tiene el usuario. Si no se ha indicado ninguno, será el que corresponde al grupo y si no se ha indicado, el informado para todos. De no encontrarse éste último, no se asumirá ningún modelo como modelo predeterminado.

En caso que existan modelos con la misma descripción en la lista de la Figura 3-Modelos definidos, se exhibe la descripción concatenada con el nombre del usuario que lo creó, encerrada entre paréntesis.

Es importante aclarar que siempre que se defina un modelo en una tarea, el sistema incorpora en forma automática, un modelo llamado "Sin modelo" que no puede ser eliminado por el usuario.

Esto tiene por objeto disponer de la posibilidad que la ventana de una tarea para la cual se han definido modelos, se presente sin datos.

Esto se debe a que si bien los modelos facilitan la operatoria, también pueden complicarla en el caso en que se hayan definido modelos para una tarea y se desee ejecutarla para otro conjunto de datos diferentes a los del modelo. En ese caso es más fácil partir de una pantalla sin datos precargados. Como obligatoriamente cada pantalla se precarga con los datos que incluye el modelo definido como "Modelo predeterminado", bastará seleccionar el modelo "Sin modelo" para que el sistema deje todos los campos de la tarea en cuestión, sin información.

Retomemos, ahora, el tema relativo a los Permisos por programa (Figura 4-Permisos por programa). Hemos dicho que para que un usuario pueda definir modelos no debe estar tildado el campo No permite dar de alta modelos.

Ahora bien, en el caso que se le permita dar de alta modelos, para cada modelo es posible indicar si el usuario, o el grupo de usuarios seleccionado, tiene permisos para usar, modificar y/o dar de baja cada uno de los modelos definidos para el programa en cuestión.

Efectivamente, al indicar un usuario o un grupo, y un programa (Figura 4-Permisos por programa), la tabla disponible en la ficha Modelos exhibe los modelos habilitados para dicho usuario.

Estos pueden corresponder a los modelos habilitados para todos los usuarios, a los modelos habilitados para el grupo o grupos a lo cual pertenece el usuario o a los modelos habilitados para ese usuario, en particular. Es por ello que la columna Permisos para exhibe, según cual sea el caso, TODOS, el nombre de un grupo (ej, GRUPO1, GRUPO 2) y/o el nombre del usuario del cual se trate.

La información correspondiente a los modelos que se han habilitado, en forma directa para el usuario puede modificarse en su totalidad.

En cambio, en los modelos que el usuario tiene habilitados por pertenecer a un grupo, o porque se han habilitado para todos los usuarios, sólo se podrá modificar el estado de la columna Por defecto.

En ambos casos el usuario puede eliminar de su lista de modelos aquellos de los que no le interese disponer siempre y cuando no sea el creador del modelo que pretende eliminar de su lista de habilitados.

En caso que un usuario haya creado un modelo sólo podrá eliminárselo a sí mismo desde la tabla correspondiente al alta de modelos, eliminado el modelo mediante el botón Borrar (Figura 2-Definición de Modelos).

En relación a la Figura 4-Permisos por programa, si por algún motivo el usuario requiere incorporar modelos a la tabla, se dispone de una ventana de búsqueda en la columna Modelo. La ventana de búsqueda mostrará todos los modelos creados por cualquier usuario en los que esté habilitado el usuario en cuestión, ya sea por haberlo habilitado en forma directa o por pertenecer a determinados grupos.
