Líneas de control de ingresos

  • Número

En esta columna el sistema actualiza en forma automática el número de línea del que se trate. Asignará un número correlativo comenzando por el 1, tanto para nuevas líneas como para las que se inserten en medio de líneas existentes cada vez que se tipea un nuevo ítem.

  • Diferencias

Esta columna incluye un check box que se ha de marcar, en forma automática, luego de haberse ejecutado el proceso de comparación entre los ítems ingresados mediante esta tarea y los ingresados originalmente en el remito de ingreso que se está controlando.

  • Item

Código del ítem a incluir en el comprobante.

Debe corresponder a un bien ingresado en el sistema.

Puede ingresarse el código del ítem o su código alternativo.

En este campo se cuenta con la opción de salir de la línea con la flecha hacia abajo del teclado. Al digitar dicho botón se indica el valor uno para el campo cantidad de esa línea (según la unidad de venta del bien) y se inserta una nueva línea para el comprobante. Esto ocurre sólo si el ítem no maneja dimensiones, contenedores, despachos, partidas, colores o números de serie. En estos casos, el cursor se posiciona en el campo cantidad de la línea tipeada ya que es necesario ingresar datos adicionales. Esta nueva opción de tipeo habilita la posibilidad de ingresar líneas de ítems con lector de código de barras de teclado.

  • Descripción

Descripción del ítem indicado en el dato anterior.

Si el ítem tiene asociada una descripción larga, se exhibirá esta. Si no, se exhibirá la otra descripción. Si el ítem no tiene descripción asociada, se solicitará el ingreso de un detalle del mismo en este momento.

  • Cantidad

Cantidad de unidades que ingresan del ítem indicado.

Aquí se presentan varias posibilidades.

Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las compras se registran en la primera unidad. Si las compras se registran en segunda unidad, al ingresar la cantidad en la segunda unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable.

Si el ítem trabaja con colores y medidas se abrirá una ventana que solicitará el ingreso de las cantidades para cada color y medida solicitada, y al finalizar el ingreso de datos en esa ventana, se completará este dato con la cantidad total de unidades indicadas para todos los colores y medidas del ítem.

Si el ítem es un contenedor, se abrirá una ventana que solicita el código del ítem que contiene (se ofrecerá el ítem indicado como default en la tabla de contenedores del bien) y la cantidad del mismo, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por la cantidad de unidades de este indicada en la definición del ítem contenedor.

Si el ítem trabaja con dimensiones, se abrirá una ventana que solicitará el ingreso de cantidades para cada dimensión posible definida para el ítem, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por todas las dimensiones del ítem.

Si el grupo de compras asociado a los bienes que ingresan lo indica, se controlan las cantidades que ingresan de modo que la entrega no se realice en forma parcial y, en caso que existan diferencias, que éstas no excedan los porcentajes de tolerancia máximos admitidos.

  • Cantidad en 2da unidad

Cantidad de unidades que ingresan del ítem indicado en la segunda unidad de medida.

Si el ítem no trabaja con doble unidad de medida, no se solicitará información en este campo.

Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las compras se registran en la segunda unidad. Si las compras se registran en primera unidad, al ingresar la cantidad en la primera unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable.

  • Despacho

Código de hasta quince caracteres que indica el número de despacho al cual ingresa el ítem indicado.

Este dato podrá ingresarse para los ítems que trabajan con despachos. Debe corresponder a un despacho definido en el sistema.

  • Partida

Código de hasta quince caracteres que indica el número de la partida a la cual ingresa el ítem indicado.

Este dato podrá ingresarse para los ítems que trabajan con partidas. Debe corresponder a una partida definida en el sistema para el ítem indicado.

  • Números de serie

Si el ítem indicado trabaja con números de serie, se abrirá una ventana que permitirá el ingreso de todos los números de serie que ingresan con el comprobante.

Se solicitará el ingreso de tantos números como cantidad de unidades ingresen del ítem.

El sistema brinda la posibilidad de ingresar estos números en forma automática definiendo un prefijo común y un rango de números para los mismos. Esto es muy útil en el caso de ingresos de grandes cantidades de mercadería seriada.

  • Centro de apropiación A

Código del centro de apropiación A asociado a la línea del comprobante.

Para cada ítem ingresado en el comprobante que tiene una tabla de posiciones contables asociadas, se verificará si la imputación contable de inventario de la posición contable o la asociada al concepto indicado en el comprobante, tienen subcuentas de tipo centro de apropiación A o ambos centros. En ese caso, se solicitará el ingreso del centro de apropiación asociado.

  • Centro de apropiación B

Código del centro de apropiación B asociado a la línea del comprobante.

Para cada ítem ingresado en el comprobante que tiene una tabla de posiciones contables asociadas, se verificará si la imputación contable de inventario de la posición contable o la asociada al concepto indicado en el comprobante, tienen subcuentas de tipo centro de apropiación B o ambos centros. En ese caso, se solicitará el ingreso del centro de apropiación asociado.

  • Negocio

Si se ingresan centros de apropiación y alguno de los centros indicados administra negocios se debe ingresar el código del negocio asociado a la línea del comprobante.

Debe indicarse el código de un negocio ya definido en el sistema y correspondiente a la empresa en la cual se ingresa el comprobante.
