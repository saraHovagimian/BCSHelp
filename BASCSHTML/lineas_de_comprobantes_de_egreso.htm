Líneas de comprobantes de egreso

ÍtemÍtem

Código del ítem a incluir en el comprobante.

Debe corresponder a un bien , servicio o ítem vario ingresado en el sistema.

Puede ingresarse el código del ítem o su código alternativo.

Esto depende de que se haya activado el parámetro de comprobantes "Exigir que el ítem esté en la lista de precios asociada al cliente en los remitos de egreso". Y que se esté registrando un remito de egreso con destino a clientes (cuando el concepto del remito es el que se ha informado en los parámetros generales como "Concepto de ventas"), el código debe figurar en la lista de precios estándar, correspondiente a la condición de venta por defecto, del cliente titular del comprobante.

En este campo se cuenta con la opción de salir de la línea con la flecha hacia abajo del teclado. Al digitar dicho botón se indica el valor uno para el campo cantidad de esa línea (según la unidad de venta del bien) y se inserta una nueva línea para el comprobante. Esto ocurre sólo si el ítem no maneja dimensiones, contenedores, despachos, partidas, colores o números de serie. En estos casos, el cursor se posiciona en el campo cantidad de la línea tipeada ya que es necesario ingresar datos adicionales. Esta nueva opción de tipeo habilita la posibilidad de ingresar líneas de ítems con lector de código de barras de teclado.

DescripciónDescripción

Descripción del ítem indicado en el dato anterior.

Si el ítem tiene asociada una descripción larga, se exhibirá esta. Si no, se exhibirá la otra descripción. Si el ítem no tiene descripción asociada, se solicitará el ingreso de un detalle del mismo en este momento.

ComprobanteComprobante

Número de comprobante al que se hace referencia en la línea del egreso de materiales.

Este campo no está disponible si en los Parámetros de Datos Generales se indicó que no se utiliza la columna de comprobantes referenciados.

Si la línea del egreso de materiales fue insertada automáticamente en el comprobante mediante la selección de uno o más comprobantes pendientes, en esta columna se exhibe el número del comprobante seleccionado, correspondiente a cada ítem.

Aquí se presenta la posibilidad de invocar un pedido, un ingreso de materiales o una factura de venta para el ítem indicado en la línea del egreso de materiales. Es decir, una vez indicado el ítem es posible pulsar la tecla F3 para buscar y seleccionar la línea del comprobante que se desea incluir en el egreso de materiales. Al pulsar F3 se abre una ventana que contiene una lista con todas las líneas de pedidos, comprobantes de ingreso de materiales y facturas de venta pendientes, ingresados para el ítem indicado. Al seleccionar una línea en esta tabla, los datos de la misma se copian a la correspondiente línea del egreso de materiales. En este caso el número de comprobante, la cantidad y demás datos correspondientes a la línea del comprobante seleccionado se copiarán a la línea desde la que se hizo la selección.

Este dato no puede ingresarse manualmente. Se completa automáticamente al seleccionar comprobantes mediante los botones de comprobantes pendientes o mediante la tecla F3.

CantidadCantidad

Cantidad de unidades que egresan del ítem indicado.

Aquí se presentan varias posibilidades según el tipo de ítem del cual se trate.

  • Si el ítem trabaja con doble unidad de medida, se solicita este campo si las ventas se registran en la primera unidad. Si las ventas se registran en segunda unidad, al ingresar la cantidad en la segunda unidad se completará este dato en forma automática. (Puede ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable.)

  • Si el ítem trabaja con colores y medidas se abre una ventana que solicita el ingreso de las cantidades para cada color y medida entregada, y al finalizar el ingreso de datos en esa ventana, se completará este dato con la cantidad total de unidades entregadas para todos los colores y medidas del ítem.

  • Si el ítem es un contenedor, se abre una ventana que solicita el código del ítem que contiene (se ofrecerá el ítem indicado como default en la tabla de contenedores del bien) y la cantidad del mismo, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por la cantidad de unidades de este indicada en la definición del ítem contenedor.

  • Si el ítem trabaja con dimensiones, se abre una ventana que solicitará el ingreso de cantidades para cada dimensión posible definida para el ítem, y luego se completará este dato con la cantidad resultante de multiplicar la cantidad indicada por todas las dimensiones del ítem.

  • Si el ítem se administra mediante números de serie y si hay en stock más de 10000 unidades, antes de ingresar a la pantalla de egresos de los números de serie, se pedirá indicar un prefijo, desde y hasta número, para acotar la cantidad y evitar inconvenientes visuales y consecuentes errores.

Cantidad en 2da unidadCantidad en 2da unidad

Cantidad de unidades que se entregan del ítem indicado en la segunda unidad de medida.

Si el ítem no trabaja con doble unidad de medida, no se solicitará información en este campo.

Si el ítem trabaja con doble unidad de medida, se solicitará este campo si las ventas se registran en la segunda unidad. Si las ventas se registran en primera unidad, al ingresar la cantidad en la primera unidad se completará este dato en forma automática y podrá ser modificado sólo si se indicó que la relación entre las unidades de medida es Variable.

DespachoDespacho

Código de hasta quince caracteres que indica el número de despacho del cual egresa el ítem indicado.

Este dato podrá ingresarse para los ítems que trabajan con despachos. Debe corresponder a un despacho definido en el sistema y debe tener disponible la cantidad de unidades del ítem que se está entregando.

Se exhibirá una lista ordenada por fecha con todos los despachos en los que ingresó el ítem y podrá seleccionarse el despacho que corresponda.

  • PartidaPartida

    Código de hasta quince caracteres que indica el número de partida de la cual egresa el ítem indicado.

    Este dato podrá ingresarse para los ítems que trabajan con partidas. Debe corresponder a una partida definida en el sistema para el ítem indicado.

    El sistema exhibirá una lista con todas las partidas en las que ingresó el ítem y podrá seleccionarse la partida que corresponda.

  • Números de serieNúmeros de serie

    Si el ítem indicado trabaja con números de serie, se abrirá una ventana que permitirá el ingreso de todos los números de serie que egresan con el comprobante.

    Se solicitará el ingreso de tantos números como cantidad de unidades egresen del ítem.

    Si se el bien administra números de serie que no se asocian a cantidades unitarias, se debe indicar los números de serie con la cantidad correspondiente del mismo a egresar.

    El sistema brinda la posibilidad de ingresar estos números en forma automática definiendo un prefijo común y un rango de números para los mismos. Además, se habilita una búsqueda con todos los números de serie registrados en los ingresos de mercadería del ítem, se permite seleccionar de estos números cuáles son los que egresan, y se informa las cantidades en stock actuales.

    Al posicionar el mouse sobre la columna de observaciones se habilita un Tooltip que muestra el stock actual en ambas unidades de medida del número de serie en cuestión.

  • Centro de apropiación ACentro de apropiación A

    Código del centro de apropiación A asociado a la línea del comprobante.

    Para cada ítem ingresado en el egreso de materiales, que tiene una tabla de posiciones contables asociadas, se verificará si la imputación contable de inventario de la posición contable o la de ventas o la asociada al concepto indicado para el comprobante, tienen subcuentas de tipo centro de apropiación A o ambos centros. En ese caso, se solicitará el ingreso del centro de apropiación asociado.

  • Centro de apropiación BCentro de apropiación B

    Código del centro de apropiación B asociado a la línea del comprobante.

    Para cada ítem ingresado en el egreso de materiales, que tiene una tabla de posiciones contables asociadas, se verificará si la imputación contable de inventario de la posición contable o la de ventas o la asociada al concepto indicado para el comprobante, tienen subcuentas de tipo centro de apropiación B o ambos centros. En ese caso, se solicitará el ingreso del centro de apropiación asociado.

NegocioNegocio

Si se ingresan centros de apropiación y alguno de los centros indicados administra negocios se debe ingresar el código del negocio asociado a la línea del comprobante.

Debe indicarse el código de un negocio ya definido en el sistema y correspondiente a la empresa en la cual se ingresa el comprobante.

Si un negocio se ha definido indicando que no puede ser imputado a otro cliente, no se permitirá invocarlo si el cliente del mismo no coincide con el de este comprobante.

IngresoIngreso

Si el egreso es para consumo, con destino a un cliente o agente, y el ítem no administran stock o es un servicio de tipo cantidad, seleccionar el  ingreso de compra correspondiente al consumo.

Orden de producciónOrden de producción

Esta columna sólo aparece en los egresos de depósito a planta.

Se muestra el prefijo-número de las orden de producción correspondiente al movimiento.

Pasando el cursor por la celda se muestra un tooltip que exhibe el prefijo-número de la orden, la tarea y el producto terminado principal de la orden de producción en cuestión.

A tener en cuentaA tener en cuenta

Si se indicó algo en los parámetros del equipo, para la lectora de barras, se habilita la posibilidad de, parados en el campo de la grilla de ingreso de ítems en la columna para indicar el código del ítem, que al oprimir Shift-F2, se abra la ventana que permite escanear un código de barras con un scanner por puerto serie.

Una vez escaneado el código de barras, el mismo será procesado considerando los parámetros definidos en el programa de parámetros de ítems en la solapa de Lectora.

Ya sea que se obtenga de dicho escaneo el código del ítem o el código del ítem y su cantidad, dichos datos serán ingresados en la grilla en la primera fila disponible (en caso de haber filas vacías en cualquier lugar de la grilla) o al final si no hay filas disponibles.

En el caso que se produzca algún error en la validación de los datos ingresados, se informará de dicha eventualidad y la ventana para escaneo del código de barras se cerrará, pudiendo volver a activarse oprimiendo Shift-F2.

Si el ítem escaneado no es un ítem de tipo cantidad la ventana de escaneo también se cerrará dejando al usuario la posibilidad de ingresar el resto de los datos relevantes del ítem.

Si el ítem escaneado es un ítem de tipo cantidad que no posee dimensiones, contenedores, ni es un código madre de colores y/o medidas, ni maneja despachos, partidas o series, no precisa el ingreso de centros de apropiación, ni tiene relación variable, la ventana de escaneo seguirá activa para poder continuar escaneando códigos de barras. En caso contrario la ventana de escaneo se cerrará, para habilitar el ingreso de los datos relevantes de dicho ítem (colores, medidas, despachos, series, etc.).

Si el escaneo no incluye el dato de la cantidad del ítem se asumirá cantidad 1 en la unidad de venta de dicho ítem.

En el caso en que se cierre la ventana de escaneo, pero el ítem no tenga dimensiones, contenedores o sea un código madre de colores y/o medidas, se considerará si se indicó una cantidad en el escaneo, en caso contrario se tomará cantidad 1 en la unidad de venta del ítem y se podrá continuar con el ingreso de los otros datos relevantes del ítem.
