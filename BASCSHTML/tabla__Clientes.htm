Clientes

Aquí se administra toda la información relativa a los clientes con los que opera la empresa.

Los clientes que se incorporan al sistema constituyen subcuentas de cuentas definidas en el plan de cuentas. Esto posibilita reflejar contablemente, en forma detallada, cada operación que los involucre, sin necesidad de habilitar una cuenta por cada cliente que se habilite en el sistema.

Los datos que deben ingresarse son los siguientes:

CódigoCódigo

Código de hasta ocho caracteres que identifica al cliente y será utilizado para individualizar al mismo en todo el sistema.

Como ejemplos podemos mencionar, C00109, 68853851, etc.

Razón socialRazón social

Razón social del cliente. Se dispone de hasta sesenta caracteres para el ingreso.

Esta información figurará en todos los informes en los que intervenga el cliente y en los comprobantes que se impriman.

Ficha GeneralesFicha Generales

Se incluyen aquí todos los datos relativos a la ubicación geográfica, números telefónicos, etc.

Nombre de fantasía

Como se sabe, es común que las empresas posean, además de su razón social, un nombre de fantasía. Muchas veces, incluso, son mas conocidas por este último que por su razón social.

El sistema prevé la posibilidad de asociar un nombre de fantasía de hasta cuarenta caracteres a cada cuenta que se incorpore al sistema.

E-mail

E-mail del cliente. Se dispone de un campo de hasta cincuenta caracteres para el ingreso. Se controla que se ingrese un E- mail válido.

Mediante este botón es posible acceder a un explorador de windows con el fin de localizar el archivo que guarda la imagen que se desea asociar al cliente.

Capturar

Mediante este botón es posible digitalizar una imagen, esto significa que, si el equipo dispone de un scanner o de una cámara digital, se pueden obtener imágenes directamente de estos dispositivos. La imagen puede ser guardada en un archivo si es que se requiere que sea utilizada por otras aplicaciones.

Para la digitalización de imágenes se debe verificar que esté instalada la aplicación Imaging que viene con el sistema operativo Windows. También es necesaria para poder ver las imágenes con formato “TIF” en las consultas del sistema.

Borrar

Sucursal

El sistema prevé la posibilidad de administrar la facturación de sucursales. Se podrá definir un cliente que actúe como casa central y otros que sean considerados como sucursales de dicha casa central.

Aquí se permite indicar si el cliente actúa como sucursal de otra cuenta corriente principal.

Si es sucursal se podrán registrar comprobantes igual que para los clientes comunes y se asociarán estos comprobantes al cliente principal que administra la sucursal.

Administrada por

Código del cliente principal que administra al cliente que se está ingresando. Este dato sólo se habilita si se indicó que el cliente es sucursal.

Debe ser un cliente ya ingresado en el sistema. Un cliente puede administrar todas las sucursales que se deseen.

Este dato puede modificarse aunque ya se hayan ingresado movimientos para la sucursal. Al hacerlo, los comprobantes de cuenta corriente que se registren de allí en más quedan asociados al nuevo cliente administrador mientras que los anteriores quedan asociados al administrador anterior.

Tabla de Condiciones de ventaTabla de Condiciones de venta

Muchas veces suele ocurrir que, para un mismo cliente, no siempre se utiliza una única condición de venta . Efectivamente, a veces la condición varía en función de las características que presenta cada operación en particular, como por ejemplo, según el volumen de la venta o los descuentos que se otorgan o el plazo de pago, etc.

Para permitir que se varíe la condición de venta al momento de registrar pedidos y facturas pero, a la vez, controlar que no se utilice ninguna no autorizada por el responsable del área de facturación, se prevé la posibilidad de asociar una lista de condiciones de venta a cada cliente.

Una de las condiciones de venta que se le asocien, seguramente la que se utilice más frecuentemente en las operaciones que se realizan con el cliente en cuestión, llevará una indicación de condición de venta por defecto. Esta será la condición que se ofrezca cada vez que el cliente se invoque como titular de un comprobante pero, en los casos en que el operador lo desee, podrá reemplazarla por cualquier otra que se encuentre en la lista de condiciones mencionada.

Las condiciones de venta que se asocian al cliente llevan asociadas además, las listas de precios con las que se operará al ingresar comprobantes del cliente con cada condición de venta habilitada para el mismo. Es obligatorio que el cliente tenga para cada condición, una lista de precios para poder registrar sus pedidos y facturas.

Esto permite manejar distintas listas de precios por cliente y al ingresar la condición de venta de cada comprobante de un cliente se determinarán las listas de precios standard, oferta y de mínimos a utilizar. Entonces, al asociar una condición a un comprobante de ventas de un cliente se estará determinando qué bienes se le pueden vender, y a qué precios.

La condición de venta, al ser vinculada a las facturas y notas de débito, determina las fechas de vencimiento y los importes correspondientes a cada vencimiento (en caso que la condición sea en cuotas).

Código

Código de condición de venta que se quiere asociar al cliente. Debe ser una condición ya definida en el sistema.

Por ejemplo, CD-Contado, 30D-30 días fecha factura.

Descripción

Denominación de la condición de venta ingresada en el campo Código.

Por Defecto

Indicación de si la condición de venta ingresada en el campo Código es la condición por defecto, es decir, la que será ofrecida en el ingreso de comprobantes del cliente.

Si al ingresar un comprobante del cliente se quiere utilizar una condición de venta diferente a la que se ofrece por defecto, ésta puede reemplazarse por otra que se encuentre definida en esta lista.

Lista estándar

La Lista estándar puede ser considerada como la lista de precios de uso habitual en las operaciones con el cliente. En ella figuran los bienes que se le venden y los precios a los cuales se le venden.

Si se desea digitar el precio de venta, directamente en el momento en que se registra el comprobante, bastará con que los bienes que se le han de vender al cliente figuren en su lista estándar con precio cero.

Se debe ingresar un código de lista de precios ya definida en el sistema y habilitada para ventas. La misma será utilizada en todos los comprobantes de venta del cliente que se ingresen con la condición de venta indicada.

Lista oferta

La Lista oferta permite efectuar excepciones a lo indicado en la lista estándar. Es útil en los casos en que se desea promocionar o liquidar un conjunto de bienes mediante su venta a un precio diferencial. Efectivamente, si un ítem figura en ambas listas a la vez, al ser invocado en un comprobante de venta, se ofrece el precio que figura en la lista oferta. Si un ítem sólo figura en esta lista, también se permite su venta y, el precio de venta, es aquel que en ella se indique.

Se debe ingresar un código de lista de precios ya definida en el sistema y habilitada para ventas. La misma será utilizada en todos los comprobantes de venta del cliente que se ingresen con la condición de venta indicada.

Lista mínima

La Lista mínima está destinada a permitir, y a la vez controlar, las desviaciones de precios que pueda digitar el operador al cargar el comprobante de ventas. Efectivamente, en aquellos comprobantes donde se permita alterar el precio de venta que ofrece el sistema (ya sea el de la lista oferta o estándar), el precio digitado no podrá ser inferior al que indique la lista mínima. Si el ítem no figura en la lista mínima el precio de venta no puede alterarse.

Se debe ingresar un código de lista de precios ya definida en el sistema y habilitada para ventas. La misma será utilizada en todos los comprobantes de venta del cliente que se ingresen con la condición de venta indicada.

Ficha Contables/Impositivos

Se ingresan aquí todos los datos destinados a determinar la actitud del cliente frente al cálculo de impuestos de los comprobantes de los cuales es titular.

Los clientes llevan incorporada su categoría frente al I.V.A. DepartamentoDepartamento

Sólo será necesario ingresar datos en esta tabla en caso que el usuario sea una empresa localizada en Uruguay y dedicada a la comercialización de comestibles y bebidas.

Allí se aplica un impuesto denominado Tasa Bromatológica o Tasa departamental a aquellos entes que se dedican a la comercialización de comestibles y bebidas.

La República de Uruguay se divide en 19 departamentos, cada uno de los cuales cuenta con un gobierno autónomo denominados Intendencias Municipales. Estas intendencias son las encargadas de la recaudación de este tributo.

En este campo debe informarse el código del departamento en el que se le hace entrega de la mercadería al cliente que se está definiendo.

Aplica Discriminado

Exento

Aplica Sin Discriminar

Si se selecciona No aplica, independientemente del departamento que se informe y de los bienes que se le facturen, no se calculará el impuesto bromatológico sobre las ventas que se le efectúen a este cliente.

Si se selecciona Aplica Discriminado, se calculará el importe correspondiente al impuesto y se exhibirá discriminado en el comprobante de venta correspondiente.

Si se selecciona Aplica sin discriminar, el impuesto se calculará pero el importe resultante se considerará que forma parte del precio y no se discrimina.

Créditos máximosCréditos máximos

La definición de cada cliente puede incluir información destinada a controlar que las deudas comerciales que éste mantiene con la empresa no excedan los límites de crédito que ésta ha establecido para él.

La deuda de un cliente con la empresa está compuesta por las facturas impagas y los documentos y cheques, directos o de terceros, que aún no se han efectivizado.

Los importes de crédito máximo informados como de deuda global, no documentada, documentada propia y documentada de terceros son controlados en forma continua, por el sistema, cuando se registran comprobantes bajo la titularidad de clientes.

En le caso del control de crédito en la facturación, si la factura referencia un pedido que fue autorizado por crédito excedido, o un egreso registrado en base a un pedido autorizado por crédito excedido, se permite la registración de la venta aunque se esté excediendo el crédito disponible.

Deuda global

Importe de deuda global que se define para el cliente. Debe estar expresado en la moneda para créditos que se indicó en los Parámetros de Cuentas Corrientes.

Al registrar comprobantes (pedidos, facturas, recibos, etc.) a nombre del cliente, se controla que su deuda (integrada por todos los elementos mencionados) nunca exceda este importe.

Ficha RutasFicha Rutas

Los clientes prevén el ingreso de información destinada a ser utilizada en los procesos destinados a la generación de lotes de facturas en forma automática. Como la Prioridad de facturación de la que ya hablamos.

Muchas empresas organizan la entrega y distribución de los productos que comercializan por zonas, estableciendo distintas áreas de entrega en función a la ubicación geográfica y domicilios de entrega de los clientes. Al respecto, si bien existen múltiples variantes y situaciones, básicamente la idea es que cada uno de los medios de transporte de los que se dispone para efectuar entregas, tiene asignado un grupo de clientes.

Apuntando a estos procesos que permiten generar simultáneamente las facturas que corresponden a un determinado circuito de entrega, el sistema prevé asociar, a cada cliente que se incorpore al sistema, una serie de Rutas . Cada ruta representa un circuito de entrega que incluye a ese cliente en su itinerario.

Luego, los procesos que permiten facturar los pedidos que aún se encuentran pendientes, permitirán seleccionar solamente aquellos que corresponden a los clientes que pertenecen a determinada ruta.

Ficha SuspendidoFicha Suspendido

En esta ficha es posible suspender al cliente que se está definiendo para realizar alguna/s operación/es en particular, o para realizar cualquier operación.

Es posible indicar que todos los comprobantes están suspendidos salvo excepciones. Estas se detallan en la Tabla Excepto los indicados. Del mismo modo es posible indicar que todos están habilitados salvo excepciones que se detallan, también, en la tabla mencionada

Al intentar ingresar comprobantes suspendidos para un cliente, aparecerá en pantalla un mensaje explicando el error y no se podrá continuar con el ingreso del comprobante.

Esto puede ser útil, por ejemplo, si en algún momento por algún motivo la empresa decide no venderle a determinado cliente. Entonces para que el sistema no permita esto, se suspende al cliente para las facturas de venta, es decir, se habilitan todos los comprobantes, excepto las facturas de venta. Se marca la opción Todos habilitados y se agrega la factura de venta en la lista de comprobantes exceptuados para el cliente. Y cuando se decide volver a venderle al cliente, se sacan las facturas de venta de la lista de comprobantes exceptuados.

Todos suspendidos

Indicación de si se desea suspender el ingreso de todos los comprobantes para el cliente que se está definiendo. No obstante podrán luego indicarse, en la tabla que sigue, si hay comprobantes que estén exceptuados de esta suspensión.

Todos habilitados

Indicación de si se desea habilitar el ingreso de todos los comprobantes para el cliente que se está definiendo. No obstante podrá luego indicarse, en la tabla que sigue, si hay comprobantes que estén exceptuados de esta habilitación, vale decir que se encuentren suspendidos.

Tabla Excepto los indicados

Esta tabla permite informar todos los tipos de comprobante que se encuentran exceptuados de la suspensión o habilitación general efectuada en alguno de los campos precedentes.

Puede ingresarse un tipo de comprobante en particular o un grupo de ellos.

Pulsando el botón Todos los comprobantes, se incluirán todos los tipos de comprobantes en la tabla.

Por otro lado, el botón Vaciar Tabla, permite quitar a todos los comprobantes de la tabla.

Código

Código del tipo de comprobante que se desea exceptuar de la suspensión o habilitación indicada más arriba para el cliente que se está definiendo.

Se permitirá el ingreso de cualquier tipo de comprobante definido en el sistema en el cual sea posible ingresar clientes (facturas de ventas, pedidos, recibos, etc).

Nombre

Descripción del tipo de comprobantes correspondiente al código informado.

Ficha EmpresasFicha Empresas

Si se trabaja con múltiples empresas, aquí se puede indicar para cada cliente que se ingresa en qué empresas podrá utilizarse.

Por default se muestra una lista con todas las empresas habilitadas para el usuario que está ingresando el cliente, y en esta lista se ofrecen marcadas como habilitadas (en color rojo) todas las empresas, de manera de no tener que acceder a esta ficha si se ingresan clientes comunes a todas las empresas.

Los clientes comunes a todas las empresas también estarán habilitados para operar con nuevas empresas que se ingresen en el sistema. Al habilitar un cliente para operar con una empresa también se habilita al mismo para operar en todas las sucursales de la empresa.

Pulsando el botón Invertir selección las empresas habilitadas se deshabilitan y las no habilitadas se marcan automáticamente como habilitadas.

Empresa de alta

Código de empresa en la cual está el usuario en el momento de ingresar el cliente en el sistema.

Un usuario solamente podrá efectuar modificaciones al cliente si se encuentra en la empresa de alta asociada al mismo.

Este dato no se ingresa manualmente. Para los clientes nuevos que se ingresen, se muestra la empresa en la cual está el usuario, y para los clientes ya existentes se muestra la empresa en la cual estaba el usuario en el momento de ingresar el cliente.

Tabla de Empresas

Esta tabla muestra todas las empresas habilitadas para el usuario que está ingresando el cliente. Recordemos que un usuario podrá tener habilitadas algunas empresas de todas las existentes y esta información podrá indicarse en el punto Usuarios .

De las empresas habilitadas para el usuario se mostrarán en color rojo las empresas en las cuales el cliente está habilitado.

Un cliente podrá habilitarse o deshabilitarse para una empresa haciendo doble click en la fila de la tabla correspondiente a la misma.

Se debe tener en cuenta que si el cliente tiene sucursales la empresa no puede deshabilitarse si alguna de dichas sucursales se encuentra habilitada en esta empresa.

Empresa

Código de la empresa en la cual el cliente que se está definiendo podrá estar o no habilitado para operar.

Nombre

Nombre de la empresa correspondiente al código informado.

Copiar empresas de casa central

Este campo se habilita solo cuando el cliente es administrado por otro. Su función es copiar las empresas habilitadas/deshabilitadas de la casa central a la tabla de empresas del cliente administrado.

Cada cliente administrado por otro solo puede esta habilitado para las empresas para las que su casa central está habilitada.

  Ficha Adjuntos

Permite adjuntar archivos externos.

Se pueden asociar múltiples documentos , se podrán agregar o eliminar cuando se deseen, e incluir comentarios en cada documento adjuntado.

El nombre de cada documento adjunto funciona como un hipervínculo, al pulsar sobre el nombre del documento es posible abrirlo, editarlo y volverlo a guardar usando las características del editor o programa que corresponda al tipo de documento o archivo.
