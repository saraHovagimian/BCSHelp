Permisos por programa (Versión XE)

El sistema brinda la posibilidad de manejar para los distintos usuarios del sistema, permisos por cada programa, es decir, dentro de un determinado proceso qué cosas puede hacer un usuario y qué cosas no. Además, mediante esta tarea se pueden indicar los modelos que cada usuario puede usar para cada programa.

La operatoria consiste en elegir un usuario del sistema, seleccionar un programa, y luego habilitar o inhabilitar los datos o modelos que se deseen.

Puede seleccionarse un usuario determinado del sistema o un grupo de usuarios. Si un usuario no tiene permisos por programa específicos, usará los del grupo de usuarios al que pertenece. Si el grupo no tiene permisos por programa específicos, el usuario tendrá habilitados todos los datos y modelos de todos los programas.

Al empezar a utilizar el sistema todos los datos y modelos de todos los programas estarán habilitados para todos los usuarios del sistema y sólo a través de este punto podrán definirse las excepciones necesarias.

Si se selecciona un programa de tipo ABM (altas, bajas y modificaciones) se podrán poner permisos de alta, baja, modificación y consulta para todo el programa en general. Si el programa no es de tipo ABM los permisos anteriores estarán deshabilitados.

Al indicar el programa se completa la ficha Datos con la lista de todos los campos posibles de ingresar o modificar en el programa, y la ficha Modelos se completa con la lista de todos los modelos definidos para ese programa.

Los campos se pueden prohibir en alta, en modificación y en consulta.

Si un campo de un programa se prohíbe en alta o en modificación, el usuario o el grupo indicado, no podrá ingresar nada en ese campo al dar altas o modificaciones y el campo aparecerá en pantalla deshabilitado.

Si un campo se prohíbe en consulta, el usuario no podrá ver el valor de ese campo ni de los campos que dependen de él al consultar un registro en el programa indicado.

Se puede asignar además a cada campo un valor Predeterminado que es un valor fijo que tendrá el dato al ingresar nuevos registros en el programa.

Los datos a ingresar son los siguientes:

Grupos/usuarios

Se muestra una lista conteniendo todos los usuarios habilitados para cada grupo de usuarios definido.

Se permite seleccionar de esta lista, el usuario para el cual se desean configurar o modificar los permisos y modelos por programa.

Seleccionar

Mediante este botón es posible seleccionar el programa para el cual se desean modificar los permisos del usuario o del grupo de usuarios indicado.

Aparecerá en una ventana el menú mediante el cual opera el usuario o grupo, y será posible buscar y seleccionar en dicho menú el programa deseado. Al seleccionar el programa se exhibirá el nombre del mismo en el dato Programa.

Alta

Este dato sólo se encuentra habilitado si se seleccionó un programa de tipo altas, bajas y modificaciones.

Permite indicar si en el programa, el usuario indicado está habilitado para dar altas, es decir, para ingresar nuevos registros. Por ejemplo, se podrá indicar si el usuario podrá ingresar un nuevo ítem, en el programa de ingreso de bienes.

Modificación

Este dato sólo se encuentra habilitado si se seleccionó un programa de tipo altas, bajas y modificaciones.

Permite indicar si en el programa, el usuario indicado está habilitado para efectuar modificaciones a la información existente. Por ejemplo, se podrá indicar si el usuario podrá cambiar el grupo de compras al que pertenece un ítem, en el programa de ingreso de bienes.

Baja

Este dato sólo se encuentra habilitado si se seleccionó un programa de tipo altas, bajas y modificaciones.

Permite indicar si en el programa, el usuario indicado está habilitado para dar bajas, es decir, para eliminar registros. Por ejemplo, se podrá indicar si el usuario podrá eliminar un ítem que se encuentra en desuso, en el programa de ingreso de bienes.

Consulta

Este dato sólo se encuentra habilitado si se seleccionó un programa de tipo altas, bajas y modificaciones.

Permite indicar si en el programa, el usuario indicado está habilitado para efectuar consultas de la información existente. Por ejemplo, se podrá indicar si el usuario podrá consultar la ubicación de un ítem en un depósito, en el programa de ingreso de bienes.

Ficha Datos

Tabla de Datos

Esta tabla permite indicar qué datos del programa se desean inhabilitar para el usuario o grupo de usuarios indicado.

Al seleccionar el programa para el cual se desean configurar los permisos, se completará esta tabla con la lista de todos los datos posibles de ingresar o modificar en el programa.

Los datos que aparezcan en esta tabla se podrán prohibir en alta, en modificación y en consulta, o se les podrá asignar un valor fijo que tendrá cada dato al ingresar nuevos registros en el programa.

Dato

Nombre de cada dato del programa seleccionado para el cual será posible indicar permisos para el usuario o grupo de usuarios indicado.

Por ejemplo, si se indica el programa de ingreso de bienes, aparecerán en esta columna, entre otros datos, la tabla de impuestos, el peso, el volumen, las dimensiones, el stock ideal, el punto de reposición, el tipo de bien, etc.

Prohibir en alta

Indicación de si el dato exhibido en la columna anterior se encuentra inhabilitado o no para el usuario, al dar altas en el programa indicado.

Esta columna no estará habilitada si el usuario no tiene permiso para dar altas en el programa, es decir, si no se marcó el campo Alta situado más arriba en la pantalla.

Por ejemplo, se podrá prohibir que un determinado usuario ingrese el subdiario asociado a un talonario, al definir nuevos talonarios en el sistema.

Prohibir en modificación

Indicación de si el dato exhibido en la columna Dato se encuentra inhabilitado o no para el usuario, al realizar modificaciones en el programa indicado.

Esta columna no estará habilitada si el usuario no tiene permiso para efectuar modificaciones en el programa, es decir, si no se marcó el campo Modificación situado más arriba en la pantalla.

Por ejemplo, se podrá prohibir que un determinado usuario cambie el subdiario asociado a un talonario, al efectuar modificaciones a los talonarios definidos en el sistema.

Prohibir en consulta

Indicación de si el dato exhibido en la columna Dato se encuentra inhabilitado o no para el usuario, al realizar consultas en el programa indicado.

Esta columna no estará habilitada si el usuario no tiene permiso para efectuar consultas en el programa, es decir, si no se marcó el campo Consulta situado más arriba en la pantalla.

Por ejemplo, se podrá prohibir que un determinado usuario vea el crédito de un cliente, al efectuar consultas a los clientes definidos en el sistema.

  • Predeterminado

Este dato es opcional y corresponde al valor fijo que tendrá el dato al ingresar nuevos registros en el programa. Cada vez que sea posible ingresar un valor al dato, se completará automáticamente el mismo con el valor que aquí se indique.

Si el dato es un campo que se tilda en pantalla, los valores posibles a ingresar son S, s, N o n. Para el resto de los campos se valida que el valor predeterminado sea del mismo tipo que el campo (carácter, fecha o número) y que no exceda la longitud permitida del campo.

Ficha Modelos

No permite dar de alta modelos

Este campo permite indicar para el usuario, o el grupo de usuarios del cual se trate, si tienen o no permiso para dar de alta modelos en el programa que se indica en el campo Programa.

Por defecto, se ofrece, destildado.

Si el usuario no tiene permiso para dar de alta modelos, en la ventana correspondiente al programa del cual se trate, al pretender grabar un nuevo modelo, el botón Grabar estará deshabilitado.

Tabla de Modelos

Esta tabla permite indicar qué modelos de los definidos para el programa seleccionado, se desean inhabilitar para el usuario o grupo de usuarios indicado.

Al seleccionar el programa para el cual se desean configurar los permisos, se completará esta tabla con la lista de todos los modelos definidos y habilitados para el programa.

Esta tabla posee columnas para indicar si el usuario o grupo seleccionado tiene permisos para usar, modificar y/o dar de baja el modelo.

Modelo

Modelo del programa seleccionado, posible de inhabilitar para el usuario o grupo de usuarios indicado.

Al indicar un usuario o un grupo y un programa, la tabla disponible en la ficha Modelos exhibe los modelos habilitados para dicho usuario.

Si se seleccionó un grupo, se muestran los modelos habilitados para ese grupo y para todos los usuarios que lo integran.

Para habilitar modelos que anteriormente hayan sido inhabilitados para el usuario o grupo, se podrán agregar los mismos a la tabla, seleccionándolos de la lista de modelos definidos para el programa.

Creador

Usuario creador del modelo.

Permiso para

Exhibe este campo, según sea el caso, TODOS, el nombre de un grupo, y/o el nombre del usuario del cual se trate.

Observaciones

Texto que se ha incorporado en el campo Comentario al darse de alta un modelo, en alguna ventana del sistema correspondiente.

Uso

Si se activa este campo, el usuario y/o grupo de usuarios puede utilizar el modelo definido en la fila de la tabla.

Modificación

Si se activa este campo, el usuario y/o grupo de usuarios puede modificar el modelo definido en la fila de la tabla.

Si el usuario no tiene permiso de modificación, se deshabilitan los campos Comentario, Modelo predeterminado, el botón Grabar y el botón Seguridad de la ventana de Modelos.

Baja

Si se activa este campo, el usuario y/o grupo de usuarios puede dar de baja el modelo definido en la fila de la tabla.

Si un usuario no tiene permiso para dar de baja el modelo, se deshabilita el botón Borrar de la ventana Modelos que corresponda.

Defecto

Indicación de un modelo por defecto, para el usuario y/o grupo de usuarios determinado.
