Proveedores

Aquí se administra toda la información relativa a los proveedores de la empresa.

Los proveedores que se incorporan al sistema constituyen subcuentas de cuentas definidas en el plan de cuentas. Esto posibilita reflejar contablemente, en forma detallada, cada operación que los involucre, sin necesidad de habilitar una cuenta por cada proveedor que se habilite en el sistema.

Los datos que deben ingresarse son los siguientes:

CódigoCódigo

Código de hasta ocho caracteres que identifica al proveedor y será utilizado para individualizar al proveedor en todo el sistema.

Como ejemplos podemos mencionar, P0015, 25388851, NESTLE, etc.

Razón socialRazón social

Razón social del proveedor. Se dispone de hasta sesenta caracteres para el ingreso.

Esta información figurará en todos los informes en los que intervenga el proveedor y en los comprobantes que se impriman.

Nombre de fantasíaNombre de fantasía

Como se sabe, es común que las empresas posean, además de su razón social, un nombre de fantasía, una marca. Muchas veces, incluso, son mas conocidas por este último que por su razón social.

El sistema prevé la posibilidad de asociar un nombre de fantasía de hasta cuarenta caracteres a cada cuenta que se incorpore al sistema.

Ficha Datos GeneralesFicha Datos Generales

Se incluyen aquí todos los datos relativos a la ubicación geográfica, números telefónicos, etc.

E-mail

E-mail del proveedor. Se dispone de un campo de hasta cincuenta caracteres para el ingreso .Se controla que se ingrese un E- mail válido.

Nuestro código de cliente

Este es el código que el proveedor nos ha asignado para identificarnos como cliente.

Se dispone de hasta diez caracteres para el ingreso.

Página Web

Dirección de la página web del proveedor. Se dispone de un campo de hasta doscientos cincuenta caracteres para el ingreso.

Requiere Carpeta de Importación

Tilde requerido para todo aquel proveedor con el cual se trabajará con Carpeta de Importación.

Sucursal

El sistema prevé la posibilidad de administrar la facturación de sucursales. Se podrá definir un proveedor que actúe como casa central y otros que sean considerados como sucursales de dicha casa central.

Aquí se permite indicar si el proveedor actúa como sucursal de otra cuenta corriente principal.

Si es sucursal se podrán registrar comprobantes igual que para los proveedores comunes y se asociarán estos comprobantes al proveedor principal que administra la sucursal.

Administrada por

Código del proveedor principal que administra al proveedor que se está ingresando. Este dato sólo se habilita si se indicó que el proveedor es sucursal.

Debe ser un proveedor ya ingresado en el sistema. Un proveedor puede administrar todas las sucursales que se deseen.

Grilla de domicilios

Examinar

Mediante este botón es posible acceder a un explorador de windows con el fin de localizar el archivo que guarda la imagen que se desea asociar al proveedor.

Mediante este botón es posible digitalizar una imagen, esto significa que, si el equipo dispone de un scanner o de una cámara digital, se pueden obtener imágenes directamente de estos dispositivos. Guardar como

Borrar

Ficha Datos ComercialesFicha Datos Comerciales

Aquí se ingresa toda la información relacionada a los bienes que comercializa el proveedor, la condición habitual con la que nos vende, el individuo de la organización responsable de efectuar la compra, etc.

Comprador

Código de comprador, es decir, del responsable de las compras que se le efectúan al proveedor.

Debe ingresarse el código de un agente de tipo Comprador ya definido en el sistema.

De este modo el sistema lleva el registro de las compras efectuadas por cada individuo responsable de comprar y, consecuentemente, refleja dicha información en todos los informes estadísticos que brinda.

Lista de precios

Este dato es opcional y permite asignar al proveedor una lista de precios para obtener de ella los precios de los productos que se le compran.

Se debe ingresar un código de lista de precios ya definida en el sistema.

En la lista de precios pueden figurar todos o algunos de los bienes que se le compran al proveedor y los precios a los cuales se compran.

Al registrar comprobantes del proveedor, el sistema ofrecerá para cada ítem que se compre el precio que figure en la lista de precios y si se desea cambiarlo por otro bastará con digitar el precio de compra que corresponda, directamente en ese momento.

Exigir ingreso de C.A.I. y fecha de vencimiento en comprobante de compras

Indicación de si se desea, o no, que el sistema efectúe un control exigiendo el ingreso del Código de Autorización de Impresión correspondiente al proveedor cada vez que se ingrese un comprobantes de compras que lo tenga como titular.

Esto control no actúa en el caso de registrar liquidaciones de compras, liquidaciones de crédito de compras y comprobantes internos de compras.

Clave de acceso

Clave de acceso por Internet.

Por ser este un campo de tipo clave de acceso, al tipear la clave se verá un asterisco por cada caracter tipeado y al finalizar la definición del proveedor, si se ingresó información en este campo se pedirá una confirmación de la clave de acceso.

Condición de compra

Código de la condición por defecto, con la que le compramos al proveedor. Debe ser una condición ya definida en el sistema.

Esta condición será ofrecida en todos las compras que se le efectúen al proveedor, pudiéndose cambiar por otra en los comprobantes que se desee.

En caso que se active el campo Permitir sólo las siguientes condiciones sólo se podrá cambiar por alguna de las condiciones indicadas en la tabla de condiciones. Si no se activa, se podrá sustituir, libremente, por cualquiera de las condiciones habilitadas en el sistema.

La condición, al ser vinculada a las facturas, determina las fechas de vencimiento y las cuotas correspondientes en cada caso.

Permitir sólo las siguientes condiciones

En caso que se active, se podrá indicar en la tabla "Condiciones permitidas", las condiciones que se habilitan para el proveedores cuestión.

Condiciones permitidas

En caso que se active el campo Permitir sólo las siguientes condiciones, en esta tabla se deben indicar las condiciones que se habilitan para el proveedores cuestión.

Estas serán las únicas condiciones que se podrán utilizar en los comprobantes de compra, cotizaciones y órdenes de compra que se registren para el proveedor en cuestión. Además, serán las únicas que el sistema ofrecerá, en el campo correspondiente, cuando se realicen búsquedas en el ingreso de los comprobantes mencionados.

Si se ha indicado una condición por defecto, ésta deberá figurar, obligatoriamente en esta tabla como una de las condiciones permitidas. Por lo cual, si el usuario no la indica en la tabla, el sistema la insertará automáticamente.

Si para un determinado proveedor no se indica ninguna condición en la tabla de "Condiciones permitidas", al registrar comprobantes que lo tengan por titular, se podrá indicar cualquier condición y, en este caso, las ventanas de búsqueda del sistema ofrecerán todas las condiciones de compra/venta que se definido.

Si no se indica ninguna condición por defecto para el proveedor, se podrá activar el campo "Permitir sólo las siguientes condiciones" y registrar un conjunto de condiciones. En este caso, cuando se ingrese un comprobante de compra para el proveedor del cual se trate, el sistema no ofrecerá ninguna condición en el campo correspondiente y permitirá seleccionar alguna de las informadas como permitidas.

Tabla de Cuentas Corrientes

Para que los asientos asociados a los comprobantes se generen en forma automática (conjuntamente con la registración de éstos), es necesario que, al incorporar un proveedor al sistema, se informen también las cuentas de control previstas para la contabilización de sus movimientos. De este modo, cuando se invoque a un proveedor como titular de una operación, automáticamente, se imputará la cuenta de control que se ha asociado al mismo. Si se han asociado varias cuentas, se ofrecerá una de ellas en particular.

Esta tabla contiene una lista con todas las imputaciones contables posibles de imputar en los comprobantes en los que se involucre al proveedor.

Cuenta

Código de imputación contable que se quiera asociar al proveedor.

Debe ser una imputación contable ya definida en el plan de cuentas y sin subcuentas o con subcuentas Proveedores.

Por ejemplo, 200-Acreedores Pesos, 110202002-Acreedores Varios.

Si se ingresa una imputación contable sin subcuentas, los comprobantes del proveedor que se registren no actualizarán su cuenta corriente, se saldarán las imputaciones contables directamente con el ingreso de los comprobantes.

Nombre

Denominación de la cuenta ingresada en el campo Cuenta.

Default

Indicación de si la cuenta ingresada en el campo Cuenta es la cuenta por defecto, es decir, la que será ofrecida en el ingreso de comprobantes del proveedor.

Si al ingresar un comprobante del proveedor se quiere efectuar la contabilización en una cuenta diferente a la que se ofrece por defecto, ésta puede reemplazarse por otra que se encuentre en la lista de cuentas asociadas al individuo.

Ficha Contables e Impositivos

Se ingresan aquí todos los datos destinados a determinar la actitud del proveedor en el cálculo de impuestos de los comprobantes de los cuales es titular.

Los proveedores llevan incorporada su categoría frente al I.V.A.. De este modo el sistema calculará, en forma automática, los impuestos correspondientes a las compras en función al tipo de bienes que conforman los comprobantes en los que participa. Esto posibilita también que el sistema controle que el tipo de comprobante que se está registrando (A o B), si la empresa usuaria del sistema es inscripta) sea el que corresponda a la categoría del titular del mismo.

También se configura el cálculo de percepciones que efectúan los proveedores y deben registrarse conjuntamente con los comprobantes que documentan la compra, y la información destinada al cálculo de las retenciones de Ingresos Brutos, IVA y Ganancias.

Tratamiento impositivoTratamiento impositivo

Código del tratamiento impositivo asociado al proveedor que se está definiendo. Debe indicarse un código de tratamiento ya definido en el sistema.

El Tratamiento Impositivo es el que determina la actitud del proveedor frente al cálculo de impuestos en los comprobantes en los que interviene. Lleva información de la categoría de I.V.A., el cálculo de percepciones que se les efectúan y el cálculo de las retenciones de IVA, Ganancias y de SUSS.

Trat. impositivo provincialTrat. impositivo provincial

Código del tratamiento impositivo provincial asociado al proveedor que se está definiendo. Debe indicarse un código de tratamiento ya definido en el sistema.

El Tratamiento Impositivo provincial es el que determina la actitud del proveedor frente al cálculo del impuestos a los ingresos brutos en los comprobantes en los que interviene. Lleva información de las categoría de retenciones y percepciones de ingresos brutos.

Ficha AtributosFicha Atributos

T

En muchos casos se hace necesario asociar determinadas características e información para categorizar o agrupar proveedores. A título de ejemplo, podemos mencionar, la zona en la que están ubicados, la línea de productos que ofrecen, etc.

Estas características a asignar, que serán propias de cada titular de una cuenta corriente, se informan al sistema mediante el uso de Atributos.

Si por ejemplo se desea incorporar el tipo de proveedor como un dato adicional asociado a los mismos, se definirá, por ejemplo, un atributo de nombre Tipo de Proveedor cuyos valores correspondan a los tipos que interese distinguir (por ejemplo, en el rubro textil, de jeans, de camisas, de camperas, etc.). Luego, al incorporar a cada proveedor al sistema, se le asociará el atributo definido y el valor que le corresponda, en función del tipo de bienes de los que nos provea.

El uso de los atributos posibilita seleccionar de distintas maneras a los proveedores que se desea incluir en los diferentes procesos e informes del sistema.

Ficha SuspendidosFicha Suspendidos

En esta ficha es posible suspender al proveedor que se está definiendo para realizar alguna/s operación/es en particular, o para realizar cualquier operación.

Es posible indicar que todos los comprobantes están suspendidos salvo excepciones. Estas se detallan en la tabla Excepto los indicados. Del mismo modo es posible indicar que todos están habilitados salvo excepciones que se detallan, también, en la tabla mencionada.

Al intentar ingresar comprobantes suspendidos para un proveedor, aparecerá en pantalla un mensaje explicando el error y no se podrá continuar con el ingreso del comprobante.

Esto puede ser útil, por ejemplo, si en algún momento por algún motivo la empresa decide no comprarle a determinado proveedor. Entonces para que el sistema no permita esto, se suspende al proveedor para las facturas de compra, es decir, se habilitan todos los comprobantes, excepto las facturas de compra. Se marca la opción Todos habilitados y se agrega la factura de compra en la lista de comprobantes exceptuados para el proveedor. Y cuando se decide volver a comprarle al proveedor se sacan las facturas de compra de la lista de comprobantes exceptuados.

Ficha EmpresasFicha Empresas

Si se trabaja con múltiples empresas, aquí se puede indicar para cada proveedor que se ingresa en qué empresas podrá utilizarse.

Por default se muestra una lista con todas las empresas habilitadas para el usuario que está ingresando el proveedor, y en esta lista se ofrecen marcadas como habilitadas (en color rojo) todas las empresas, de manera de no tener que acceder a esta ficha si se ingresan proveedores comunes a todas las empresas.

Los proveedores comunes a todas las empresas también estarán habilitados para operar con nuevas empresas que se ingresen en el sistema. Al habilitar un proveedor para operar con una empresa también se habilita al mismo para operar en todas las sucursales de la empresa.

Pulsando el botón Invertir selección las empresas habilitadas se deshabilitan y las no habilitadas se marcan automáticamente como habilitadas.

Un usuario solamente podrá efectuar modificaciones al proveedor si se encuentra en la empresa de alta asociada al mismo.

Este dato no se ingresa manualmente. Para los proveedores nuevos que se ingresen, se muestra la empresa en la cual está el usuario, y para los proveedores ya existentes se muestra la empresa en la cual estaba el usuario en el momento de ingresar el proveedor.

De las empresas habilitadas para el usuario se mostrarán en color rojo las empresas en las cuales el proveedor está habilitado.

Un proveedor podrá habilitarse o deshabilitarse para una empresa haciendo doble click en la fila de la tabla correspondiente a la misma.

El proveedor no se puede deshabilitar en una empresa si posee sucursales que están habilitadas para dicha empresa.

Copiar empresas de casa central

Este campo se habilita sólo cuando el proveedor es administrado por otro. Se utiliza para copiar las empresas habilitadas/deshabilitadas de la casa central a la tabla de empresas del proveedor administrado.

Cada proveedor administrado por otro sólo puede estar habilitado para las empresas para las que su casa central está habilitada.

  Ficha Adjuntos

Permite adjuntar archivos externos.

Se pueden asociar múltiples documentos , se podrán agregar o eliminar cuando se deseen, e incluir comentarios en cada documento adjuntado.

El nombre de cada documento adjunto funciona como un hipervínculo, al pulsar sobre el nombre del documento es posible abrirlo, editarlo y volverlo a guardar usando las características del editor o programa que corresponda al tipo de documento o archivo.
